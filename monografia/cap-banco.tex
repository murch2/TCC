\subsection{Jogador}

	A tabela Jogador armazena os dados dos usuários utilizados pelo jogo.
	
\begin{table}[!h]
\large
\centering
\begin{tabular}{|c|c|c|c|c|}

\hline
\underline{id} & nome & moedas & epeciais & foto \\
\hline

\end{tabular}
\caption{Tabela Jogador}
\end{table}

\begin{itemize}
\item \textbf{id} - É a chave primária da tabela, se trata do id do facebook do jogador.
\item \textbf{nome} - Nome completo do jogador, informação retirada do perfil no facebook.

\item \textbf{epeciais} - Quantidade de especiais que o jogador possui.
\item \textbf{foto} - Url da foto de perfil do facebook. 
\end{itemize}

\subsection{Desafios}

	A tabela de desafios representa os dois últimos turnos jogados por dois jogadores que estão competindo. Nesta 
tabela, o jogador 1 representa o desafiante e o jogador 2 o desafiado. Como a cada turno o desafiado manda
um jogo de volta para o desafiante, este torna-se desafiado e aquele o desafiante, alternando assim os papéis. 
Por esse estilo de jogo cada partida possui duas tuplas na tabela desafios, uma que representa o turno de ida, 
e outra, que possui chave primária trocada, ou seja o jogador 1 vira jogador 2 e vice-versa, que representa o
turno de volta. 

\begin{table}[!h]
\large
\centering
\begin{tabular}{|c|c|c|c|
c|c|}

\hline
\underline{id\_jogador1} & \underline{id\_jogador2} & id\_carta & pontuacao1 \\ 
\hline
pontuacao2 & status \\
\cline{1-2}

\end{tabular}
\caption{Tabela Desafios}
\end{table}

\begin{itemize}
\item \textbf{id\_jogador1} - Faz parte da chave primária da tabela, representa o id do jogador desafiante. 
\item \textbf{id\_jogador2} - Faz parte da chave primária da tabela, representa o id do jogador desafiado. 
\item \textbf{id\_carta} - Representa a carta utilizada no último turno que o jogador desafiante enviou para o desafiado. 
\item \textbf{pontuacao1} - Quantidade de pontos obtidos pelo desafiante no turno corrente.
\item \textbf{pontuacao2} - Quantidade de pontos obtidos pelo desafiado no turno corrente.
\item \textbf{status} - Trata-se de um inteiro entre 0 e 4 que representa o estado do turno. 

\begin{itemize}
\item \textbf{0}: Representa que nenhum dos jogadores jogou este turno.
\item \textbf{1}: Representa que o desafiante está jogando.
\item \textbf{2}: Representa que o desafiante já enviou o desafio.
\item \textbf{3}: Representa que o desafiado está jogando.
\item \textbf{4}: Representa que o desafiado já respondeu o desafio.
\end{itemize}
\end{itemize}

\subsection{Historico\_Jogo}

	A tabela Historico\_Jogo guarda o placar total do jogo, fazendo a totalização de todos os turnos desde
o primeiro. Nesta tabela o id do jogador 1 é sempre numericamente menor do que o do jogador 2, fazendo com 
que a consulta SQL utilizada para a recuperação dos dados seja feita de maneira mais simples.

\begin{table}[!h]
\large
\centering
\begin{tabular}{|c|c|c|c|}

\hline
\underline{id\_jogador1} & \underline{id\_jogador2} & vitorias1 & vitorias2 \\
\hline

\end{tabular}
\caption{Tabela Historico\_Jogo}
\end{table}

\begin{itemize}
\item \textbf{id\_jogador1} - Faz parte da chave primária da tabela, representa um dos jogadores. 
\item \textbf{id\_jogador2} - Faz parte da chave primária da tabela, representa um dos jogadores. 
\item \textbf{vitorias1} - Quantidade de vitórias totais do jogador 1.
\item \textbf{vitorias2} - Quantidade de vitórias totais do jogador 2.
\end{itemize}

\section{Arquivos de criação e de inserção}
\label{sec:arquivos}

	Foram criados dois arquivos com extensão SQL para fazer a criação das tabelas do 
banco de dados: 
\begin{itemize}
\item{\textbf{ModeloFisico}} - O arquivo ModeloFisico.sql executa a criação de toda a estrutura (tabelas, chaves primárias, chaves estrangeiras, etc) presente no banco de dados de Biografia.

\item{\textbf{Populate}} - O arquivo Populate.sql popula o banco de dados com uma quantidade de dados suficiente
para que alguns testes envolvendo jogadores diferentes, e cartas diferentes possam ser executados.

\end{itemize} 

	Ambos os arquivos podem ser executados através da linha de comando. Dentro do SGBD PostgreSQL basta digitar:

``\textbackslash i Nome\_do\_arquivo'' para que o mesmo seja executado.
